Submission checklist

Make the endpoint easy for agents to trust.

The listing should help a buyer agent answer the practical questions before payment: what action happens, how much risk exists, what proof is returned, and how to verify it.

1

Describe the side effect

State the exact public HTTPS action: notify, dispatch, ingest, update, or alert.

2

Provide a safe example payload

Use placeholders for target secrets and avoid private customer, wallet, or API data.

3

Include a proof or test path

Link to an Action402 proof badge, recent proof, or a free canary/preflight result.

4

Declare limits

Document rate limits, idempotency expectations, supported methods, and failure behavior.
